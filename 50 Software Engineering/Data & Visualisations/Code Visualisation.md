# Resources

- [appmap (applandinc) (14 ★)](https://github.com/applandinc/appmap) - Allows you to build interactive stack trace maps from your code
- [Archsense | Architecture visualization](https://www.archsense.dev/)
- [django-silk (jazzband) (3.2k ★)](https://github.com/jazzband/django-silk) - Silky smooth profiling for Django
- [gprof2dot (jrfonseca) (2.3k ★)](https://github.com/jrfonseca/gprof2dot) - Converts profiling output to a dot graph.
- [madge (pahen) (5.8k ★)](https://github.com/pahen/madge) - Create graphs from your CommonJS, AMD or ES6 module dependencies
- [pyan (davidfraser) (398 ★)](https://github.com/davidfraser/pyan) - pyan is a Python module that performs static analysis of Python code to determine a call dependency graph between functions and methods. This is different from running the code and seeing which functions are called and how often; there are various tools that will generate a call graph in that way, usually using debugger or profiling trace hooks - for example
- [Gource (acaudwell) (9k ★)](https://github.com/acaudwell/Gource/) - software version control visualization
- [How do you visualize code? · Caffeinspiration](https://alexanderell.is/posts/visualizing-code/)
- [snakefood: Python Dependency Graphs](https://furius.ca/snakefood/)
- [Dependency Graphs | Understand Legacy Code](https://understandlegacycode.com/dependency-graphs/)
- [Python Object Graphs — objgraph 3.5.0 documentation](https://mg.pov.lt/objgraph/)
- [git-of-theseus (erikbern) (2.3k ★)](https://github.com/erikbern/git-of-theseus) - Analyze how a Git repo grows over time
