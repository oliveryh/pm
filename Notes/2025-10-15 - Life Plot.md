# Introduction

I have a mental model of how all of my devices, code, processes all interconnect and interact. What I’ve been feeling recently is that that web of process and possession is growing too unwieldy - it’s getting distracting.

To understand the extent of how bad things might be, and visualise how things are connected, I think I can use Obsidian. There are two reasons for this:

- I’m committed to the file over app principal, and have seen some benefits from syncing some online databases into files
- The plugin system improves the likelihood I can simply create or amend to suit my needs

The tool needs to be able to do the following

- Show nodes for things that may not have a relationship
- Be able to name edges between nodes to identify why they’re linked
- Ideally use edge styles to indicate if the link between nodes is relating to an automated or manual process, and if it’s a process that should be removed

Through this being a graph, it should also be possible to identify long drawn out processes and prune/contract the graph.
