# Annotating By Blocks

One idea that I've thought of is to annotate by blocks. This is mainly to make it easier to save the markdown highlight.

[remarkable (jonschlinkert) (5.4k ★)](https://github.com/jonschlinkert/remarkable) appears to add data row numbers to the html. It might be possible in this instance to collect these numbers and use them to store/send the appropriate markdown

[markdown-it-source-map (tylingsoft) (11 ★)](https://github.com/tylingsoft/markdown-it-source-map) - Add source map to the output HTML of markdown-it. *This could also be used as a way to map markdown to html output*
