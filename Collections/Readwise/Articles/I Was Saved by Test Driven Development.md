## Highlights

- I'd done training courses like [Code Retreats](https://www.coderetreat.org/) facilitated by fantastic software engineers to practice TDD. ([View Highlight](https://read.readwise.io/read/01gs4x2jt1nx40eej56s9ct0rg))
- I started with writing a TODO list of tests which I felt I would need. In this case, it was just unimplemented tests right in the source code. Luckily it doesn't take much sustained focus to write a list like this. You can jot them down in any order, and keep adding to the list later as new ideas occur.
